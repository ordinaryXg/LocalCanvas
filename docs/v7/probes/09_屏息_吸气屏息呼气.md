# 探针 09 — 屏息：吸气 · 屏息 · 呼气

> **概念锚点**：共同沉默后再共同看见；克制「呕吐式输出」。  
> **实验 E5**：与探针 08 共用满意度指标。

---

## 一、交互稿

### 1.1 呼吸三相

| 相 | 用户侧 | 系统侧 | UI |
|----|--------|--------|-----|
| **吸气 inhale** | 输入碎片、拖入源、哼唱 | 记录输入，不生成 | 边缘柔和脉动 inward |
| **屏息 hold** | 停顿 ≥3s 或主动按 `H` | 展开内部候选；跑预演 | 中央细线 + 「……」 |
| **呼气 exhale** | 查看幽灵/轻提案 | 展示候选 | 向外扩散动画 |

### 1.2 状态机

```
idle ──input──► inhale ──3s无输入──► hold ──probe done──► exhale ──user确认/超时──► idle
                  ▲                      │
                  └──── 新输入重置 ────────┘
```

**硬规则**：`inhale →` 禁止弹出 WorkflowPlan 全屏；最多 toast「已记下」。

### 1.3 主动屏息

- 快捷键 `H`：强制进入 `hold`，显示「一起等一等」
- 麦克风可选：检测到 2s 沉默也可触发（默认关）

### 1.4 呼气产物层级

| 级别 | 内容 |
|------|------|
| L0 | 仅文字方向摘要 |
| L1 | 幽灵预览（探针 08） |
| L2 | 用户点击升格正式生成 |

默认 L1；预算不足降 L0。

## 二、数据模型

```typescript
type BreathPhase = 'idle' | 'inhale' | 'hold' | 'exhale'

interface BreathState {
  phase: BreathPhase
  inhaleStartedAt: number | null
  holdStartedAt: number | null
  /** 本周期收集的输入事件 id */
  inhaleEventIds: string[]
}
```

纯前端 Zustand 即可；可选 telemetry 表统计周期时长。

## 三、技术探针

### 3.1 MVP（2 天）

1. `breathStore` + `BreathOverlay.tsx`（CSS 动画）
2. 全局 `useInhaleCapture` 钩子：
   - 监听 resonance patch、phrase blur、地形 mouseup
3. `useBreathTimer`：inhale 后 3s → `hold` → 发 IPC `probe:schedule`
4. **拦截**：`AgentPanel` 自动规划改到 `exhale` 相才允许（feature flag）

### 3.2 组件

```
src/components/breath/BreathOverlay.tsx
src/stores/breathStore.ts
src/hooks/useBreathCycle.ts
```

### 3.3 动画规格

```css
/* inhale: 边框 box-shadow 内缩 */
/* hold: 1px 水平线，opacity 0.6 呼吸 4s 周期 */
/* exhale: 幽灵卡片 scale 0.92 → 1 */
```

### 3.4 与 Agent 整合
