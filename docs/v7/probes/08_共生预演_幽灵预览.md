# 探针 08 — 共生预演：幽灵预览

> **概念锚点**：AI 在用户犹豫时低分预演；升格才正式渲染。  
> **实验 E5**：开启预演用户满意度 +15% vs 对照（简问卷）。

---

## 一、交互稿

### 1.1 触发条件（注意力共享）

| 信号 | 权重 |
|------|------|
| 共鸣场 `gravity` 变化 | 高 |
| 地形画笔停顿 >2s | 中 |
| 文本 phrase 编辑停止 3s | 高（屏息联动，探针 09） |
| 合唱团 resolution 应用 | 高 |

满足 → 进入 **预演队列**（debounce 3s）。

### 1.2 幽灵预览呈现

- 位置：画布**右下 240×135** 毛玻璃卡片
- 内容：低分辨率静帧或 2s 循环 mp4（`assets/.probe/`）
- 样式：15% 边框发光，标注「幽灵 · 非最终」
- 交互：
  - **点头**（点击）→ 升格正式生成
  - **摇头**（拖出屏幕）→ 丢弃 + 记录负反馈
  - **忽视** 30s → 自动淡出

### 1.3 预演池偏转

用户改共鸣场时，池中 3 个方向向量插值偏转（UI：卡片角标 1/2/3 微切换）。

### 1.4 算力契约（对用户可见）

设置 → 预演：

```
每日幽灵预算： [ 20 ] 次
仅在我停顿后出现： [x]
```

超预算 → 只显示文字方向摘要，不出图。

## 三、技术探针

### 3.1 MVP（4 天）

1. **`ProbeScheduler`**（main process）
   - 监听 `resonance:patched` IPC（探针 02 发出）
   - debounce 3s → 若 budget OK → 排队

2. **低分生成**
   - 图片：512px、steps=4、同一模型
   - 视频：仅首帧 512 或 2s 低 fps 通过现有 adapter `quality: draft`

3. **`GhostPreviewCard.tsx`** 浮层

4. **升格**：`promote(id)` → 用同 `compiledPrompt` 调正式 generate

### 3.2 挂接

```typescript
// electron/utility/services/probe/probe-scheduler.ts
export class ProbeScheduler {
  private timer: NodeJS.Timeout | null = null
  onResonanceChange(projectId: string) {
    clearTimeout(this.timer!)
    this.timer = setTimeout(() => this.runProbe(projectId), 3000)
  }
}
```

Renderer 在 `ResonancePanel` patch 后 `window.api.probe.notifyResonanceChange()`.
