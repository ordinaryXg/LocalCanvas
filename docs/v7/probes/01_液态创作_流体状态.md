# 探针 01 — 液态创作：流体状态

> **概念锚点**：作品不是静态图，而是带温度/粘度/表面张力的可流动介质；导出是结晶，不是终点。  
> **假设**：用户能通过「冷热/稀稠」直觉理解项目成熟度，而不需要理解节点图。

---

## 一、交互稿

### 1.1 首屏：雾池（Fog Pool）

**触发**：打开项目 / 新建项目后进入「液态视图」（默认，可下潜引擎视图）。

| 元素 | 行为 |
|------|------|
| 中央雾层 | WebGL/Canvas 噪声渐变；色相向 `fluid.temperature` 偏移（热=暖橙，冷=蓝灰） |
| 余温提示 | 上次关闭时间 < 24h 时，雾心微亮；文案：「项目仍热」 |
| 掬取手势 | 长按雾面 0.6s → 弹出最近 3 个「未结晶镜头」缩略圆环 |
| 相位指示器 | 右下角三点：探索 / 收敛 / 冻结（见探针 11） |

### 1.2 流体参数调节（非表单）

三枚**物理滑块**（竖向，像混音台推子）：

| 推子 | 映射 | 左端 | 右端 |
|------|------|------|------|
| 温度 | `temperature` 0–1 | 冷（已敲定多） | 热（大量探索中） |
| 粘度 | `viscosity` 0–1 | 稀（易改） | 稠（改动会警告） |
| 张力 | `surfaceTension` 0–1 | 场景融在一起 | 场景边界锐利 |

**交互规则**：

- 拖动粘度 → 稠时尝试删除已「结晶」片段需二次确认
- 温度自动计算为主、手调为辅（见数据模型）
- 张力影响共鸣场行星轨道间距（探针 02）

### 1.3 状态转换

```
[探索] --生成首个镜头--> [收敛]
[收敛] -->50% 镜头结晶--> [冻结倾向]
任意 --导出 MP4--> 结晶快照 + 流体仍保留
```

## 二、数据模型

```typescript
/** 项目级流体状态 — 存 SQLite fluid_state 表 */
interface FluidState {
  projectId: string
  /** 0=冷 1=热；公式见下 */
  temperature: number
  viscosity: number
  surfaceTension: number
  /** explore | converge | freeze */
  phase: 'explore' | 'converge' | 'freeze'
  lastSessionEndedAt: string | null
  /** 结晶镜头 id 列表（不等于删除，仍可回熔） */
  crystallizedShotIds: string[]
  updatedAt: string
}

/** 温度自动估算（每小时衰减） */
function computeTemperature(input: {
  openSuperposedCount: number
  recentEditCount24h: number
  crystallizedRatio: number
}): number {
  const heat = input.openSuperposedCount * 0.15 + input.recentEditCount24h * 0.05
  const cool = input.crystallizedRatio * 0.6
  return Math.max(0, Math.min(1, 0.3 + heat - cool))
}
```

**SQLite**

```sql
CREATE TABLE fluid_state (
  project_id TEXT PRIMARY KEY,
  temperature REAL NOT NULL DEFAULT 0.5,
  viscosity REAL NOT NULL DEFAULT 0.3,
  surface_tension REAL NOT NULL DEFAULT 0.5,
  phase TEXT NOT NULL DEFAULT 'explore',
  last_session_ended_at TEXT,
  crystallized_shot_ids TEXT NOT NULL DEFAULT '[]',
  updated_at TEXT NOT NULL
);
```

## 三、技术探针

### 3.1 MVP 范围（2 天）

1. 新建 `FluidFogView.tsx` 全屏覆盖层，用 CSS `filter` + 动画噪声 PNG 模拟雾（无需 WebGL）
2. `fluidStore` + IPC `fluid:getState` / `fluid:patchState`
3. 打开项目时读 `fluid_state`；无记录则初始化
4. 三推子绑定 Zustand；`temperature` 每 30s 根据叠加态数量重算
5. **不实现**真实流体物理仿真

### 3.2 复用现有模块

| 模块 | 用法 |
|------|------|
| `canvasStore` | 统计节点数、最近 `updatedAt` |
| `history` IPC | `recentEditCount24h` |
| 探针 03 `superposed_shots` | `openSuperposedCount` |

### 3.3 建议文件

