## 文档结构约定

每份探针文档包含：

| 章节 | 内容 |
|------|------|
| **概念锚点** | 一句话 + 要证伪的假设 |
| **交互稿** | 用户路径、手势、状态、线框、错误态 |
| **数据模型** | TypeScript 接口 + SQLite 表草案 |
| **技术探针** | 最小实现路径、复用现有模块、IPC 草案 |
| **探针代码入口** | 建议新增文件路径与 1–3 天可跑通的 MVP 范围 |
| **验收** | 可观测指标 + 失败降级 |

## 全局技术约定

- **用户层状态**：`FluidProjectState`（新 SQLite 表 + Zustand `fluidStore`）
- **引擎层**：继续复用 `canvasStore` 节点图；由 `fluid-compiler` 双向同步
- **AI 调用**：探针阶段统一走现有 `electron/main/ipc/agent.ts` + 已配置 LLM
- **媒体**：仍落盘到项目 `assets/`；叠加态/幽灵预览用低分辨率缓存目录 `assets/.probe/`
