# 探针 04 — 情感地形：可行走地图

> **概念锚点**：情感历程是三维地形；生成前模拟「情感物理」。  
> **实验 E3**：用地形完成节奏规划的用户，分镜→首帧生成路径比对照组少 2 步。

## 一、交互稿

### 1.1 坐标系

| 轴 | 映射 | UI |
|----|------|-----|
| X | 时间 0 → 总时长 | 左右 |
| Y | 唤醒度 arousal 0–1 | 高度 |
| Z | 效价 valence -1 → +1 | 颜色（冷蓝 ↔ 暖金） |

### 1.2 编辑手势

| 手势 | 效果 |
|------|------|
| 画笔抬高 | 在 X 区间刷出 arousal 包络 |
| 捏合 | 缩放时间窗口 |
| 放置锚点 | 关键情绪时刻（可绑 phrase） |
| 行走预览 | 拖播放头小人沿地形 X 移动；脚下显示当前 (a,v) |

### 1.3 AI 建议：悬崖检测

当地形一阶导超过阈值：

```
⚠ 12.4s 处情感落差 Δarousal=0.62 / 0.8s
  [ 插入 1.2s 黑场 ]  [ 插入缓冲镜头 ]  [ 忽略 ]
```

建议写入 `terrain_suggestions` 表，用户采纳后生成对应节点（经探针 12 编译）。

### 1.4 与镜头关系

- 每个 `shotSlot` 在地形上占 `[t0, t1]` 区间
- 镜头默认 arousal = 区间平均高度
- 生成 prompt 注入：`emotional context: arousal=0.7, valence=-0.3`

## 二、数据模型

```typescript
interface AffectAnchor {
  id: string
  timeSec: number
  arousal: number
  valence: number
  label?: string
}

interface AffectEnvelope {
  projectId: string
  durationSec: number
  /** 每秒采样，长度 = durationSec * sampleRate */
  sampleRate: 2
  arousalSeries: number[]
  valenceSeries: number[]
  anchors: AffectAnchor[]
}
```

**SQLite**：`affect_envelope` 表 JSON 存 series；或拆 `affect_samples(project_id, t, arousal, valence)`。

### 3.2 组件

```
src/components/affect/AffectTerrainEditor.tsx  // 曲线编辑
src/components/affect/CliffDetector.ts         // 纯算法
electron/utility/services/affect/cliff-detector.ts
```

**悬崖算法**：

```typescript
export function detectCliffs(series: number[], sampleRate: number, threshold = 0.4) {
  const cliffs: { timeSec: number; delta: number }[] = []
  for (let i = 1; i < series.length; i++) {
    const dt = 1 / sampleRate
    const slope = (series[i] - series[i - 1]) / dt
    if (Math.abs(slope) > threshold) cliffs.push({ timeSec: i / sampleRate, delta: series[i] - series[i-1] })
  }
  return cliffs
}
```

### 3.3 生成注入

```typescript
// 在 compilePrompt 前
function affectAtTime(envelope: AffectEnvelope, t: number) {
  const idx = Math.floor(t * envelope.sampleRate)
  return { arousal: envelope.arousalSeries[idx], valence: envelope.valenceSeries[idx] }
}
```
