# 探针 12 — 引擎地下室：投影与编译

> **概念锚点**：液态体验 ↔ 节点 DAG 双向翻译；用户默认只见客厅。  
> **优先级**：**最先实现** — 其余探针都依赖本层。

---

## 一、架构

```
┌─────────────────────────────────────┐
│  Fluid Layer (用户层)                  │
│  resonance, affect, superposed, ...   │
└──────────────┬──────────────────────┘
               │ fluid-compiler
               ▼
┌─────────────────────────────────────┐
│  Engine Layer (现有)                   │
│  canvasStore nodes/edges, DAG, FFmpeg  │
└─────────────────────────────────────┘
```

**原则**：

- 用户操作流体 → `compileDown()` 增量更新节点
- 用户在地下室改节点 → `projectUp()` 同步流体摘要
- 冲突：流体优先（可配置）

## 三、核心 API

```typescript
// electron/utility/services/fluid-compiler/index.ts

export interface FluidProjectState {
  fluid: FluidState
  resonance: ResonanceField
  affect: AffectEnvelope | null
  bindings: ShotSlotBinding[]
}

/** 用户层 → 引擎：增量编译 */
export async function compileDown(
  state: FluidProjectState,
  canvas: CanvasGraph,
  changed: ('resonance' | 'affect' | 'chorus')[]
): CanvasGraph

/** 引擎 → 用户层：投影摘要 */
export function projectUp(canvas: CanvasGraph): Partial<FluidProjectState>
```

### 3.1 compileDown 示例（共鸣场）

```typescript
function compileResonance(field: ResonanceField, graph: CanvasGraph) {
  const { prompt } = compilePrompt('', field)
  let textNode = findOrCreateNode(graph, 'intent-compiler-text')
  textNode.data.draft = prompt
  textNode.data.outputMode = 'draft'
  // 参考图：每个 blur_image 源 → ImageNode 连线到下游 VideoNode
  return graph
}
```

### 3.2 projectUp 示例

```typescript
function projectUp(graph: CanvasGraph) {
  const videos = graph.nodes.filter(n => n.type === 'video')
  return {
    bindings: videos.map(v => ({
      shotSlotId: `videoNode:${v.id}`,
      nodeId: v.id,
      nodeType: 'video',
    })),
  }
}
```

## 四、同步策略

| 事件 | 方向 |
|------|------|
| 调谐盘改引力 | compileDown(resonance) |
| 地下室改 VideoNode prompt | projectUp → 警告「与共鸣场不一致」 |
| DAG 运行完成 | projectUp + 更新 superposed |
| 导出 | 只读 compile 快照 |

**冲突 UI**：

```
共鸣场与节点 prompt 不一致
  [ 以调谐盘为准 ]  [ 以节点为准 ]  [ 保留两者，标记分叉 ]
```
