## 3. 状态机

```
                    ┌─────────────┐
     resonance PATCH│             │engine SAVE
                    ▼             ▼
              ┌─────────────────────────┐
              │      in_sync            │
              └─────────┬───────────────┘
                        │ hash mismatch
                        ▼
              ┌─────────────────────────┐
              │      conflict           │
              └─────────┬───────────────┘
          resolve tuning│         │resolve engine
                        ▼         ▼
                   in_sync    forked（两源并存）
```

**forked**：`VideoNode.data.fluidForked = true`，生成时用节点 prompt，不调谐盘。

### 6.4 compileResonance 伪代码

```typescript
export function compileResonance(
  field: ResonanceField,
  nodes: CanvasNode[],
  edges: CanvasEdge[],
  projectId: string,
): CompileDownResult {
  const { prompt, negativePrompt } = compilePrompt('', field)
  const { nodes: n2, intentId } = upsertIntentNode(nodes, projectId, prompt)
  let n3 = n2, e2 = edges
  for (const src of field.sources.filter(s => s.type === 'blur_image' && s.assetPath)) {
    ({ nodes: n3, edges: e2 } = upsertRefImageNode(n3, e2, projectId, src))
  }
  return { nodes: n3, edges: e2, changedNodeIds: [intentId] }
}
```

## 8. 依赖、风险、降级

| 风险 | 对策 |
|------|------|
| 隐藏节点被用户误删 | 地下室 intent 节点 `deletable: false` |
| 编译覆盖手改 prompt | 默认冲突条，不静默覆盖 |
| 大图参考重复导入 | assetPath 去重，同 path 不建第二 ImageNode |

**降级**：关闭 `FLUID_RESONANCE` 时整个编译器 idle，canvas 纯手动。
