# 探针 10 — 结晶：流体冻结与导出

> **概念锚点**：导出是拍照，不是抽干流体；项目导出后仍可继续编辑。  
> **实验 E7**：导出后 7 日内同项目继续编辑率 >60%。

---

## 一、交互稿

### 1.1 结晶 vs 导出

| 动作 | 流体 | 时间线 | 叠加态 |
|------|------|--------|--------|
| **局部结晶** | 粘度↑ | 锁定片段 | 坍缩为 primary |
| **大规模结晶** | 温度↓ | 全片锁定 | 未坍缩提示处理 |
| **导出 MP4** | 生成快照 | 只读副本 | 快照含 primary |

导出完成文案：**「已拍照 · 流体仍在」** + 按钮「继续沸腾」。

### 1.2 导出前检查

```
结晶检查清单
  ✓ 12/12 镜头已坍缩
  ⚠ 2 个鬼影未处理  [ 归档全部 ] [ 返回处理 ]
  ✓ 情感地形无未处理悬崖
  → [ 导出 47s MP4 ]
```

### 1.3 结晶快照

每次导出写入 `crystallization_snapshots`：

- `mp4Path`
- `fluidState` JSON 副本
- `composeGraph` 副本
- `exportedAt`

**回滚**：「回到导出前流体」→ 恢复 fluid + compose，**不删**导出文件。

### 1.4 局部结晶手势

时间线片段右键 → **结晶此段** → 片段边框结冰图标；拖动需确认「回熔」。

## 二、数据模型

```typescript
interface CrystallizationSnapshot {
  id: string
  projectId: string
  mp4Path: string
  durationSec: number
  fluidState: FluidState
  composeState: unknown          // 现有 compose clip JSON
  superposedPrimaryMap: Record<string, string>  // slotId → candidateId
  createdAt: string
}

interface CrystallizedSegment {
  clipId: string
  projectId: string
  crystallizedAt: string
}
```

**SQLite**

```sql
CREATE TABLE crystallization_snapshots (
  id TEXT PRIMARY KEY,
  project_id TEXT NOT NULL,
  mp4_path TEXT NOT NULL,
  payload TEXT NOT NULL,
  created_at TEXT NOT NULL
);
```

## 三、技术探针

### 3.1 MVP（2 天）

1. 包装现有 `ComposeExportDrawer` 导出成功回调
2. 导出前调用 `superposed:allCollapsed?` 检查
3. 导出后 `fluid_state.temperature -= 0.2`（不低于 0.1）
4. 写 snapshot 行
5. StartPage 项目卡显示「最近结晶：6-6 22:10」

### 3.2 复用

| 模块 | 用法 |
|------|------|
| `useCompose` | 取时间线 JSON |
| `ffmpeg-service` | 现有导出 |
| `fluid-state-service` | 读写信 |

### 3.3 IPC

```
crystallize:precheck
crystallize:exportAndSnapshot
crystallize:listSnapshots
crystallize:restoreSnapshot
```
