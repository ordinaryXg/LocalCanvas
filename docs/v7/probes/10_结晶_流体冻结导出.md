## 3. 状态机

```
export_click --> precheck --> (issues?) user_resolve --> exporting --> success_snapshot
success_snapshot --> fluid_cool (temp -= 0.2, viscosity += 0.15)
```

**回滚**：`restoreSnapshot(id)` → 恢复 payload 内 fluid + compose clips，不删 mp4。

## 4. 数据模型

```typescript
interface CrystallizationSnapshot {
  id: string
  projectId: string
  mp4Path: string
  durationSec: number
  payload: {
    fluidState: FluidState
    composeNodeId: string
    clips: ComposeClipItem[]
    superposedPrimaryMap: Record<string, string>
  }
  createdAt: string
}

interface CrystallizedSegment {
  clipId: string
  projectId: string
  crystallizedAt: string
}
```

```sql
CREATE TABLE crystallization_snapshots (
  id TEXT PRIMARY KEY,
  project_id TEXT NOT NULL,
  mp4_path TEXT NOT NULL,
  duration_sec REAL NOT NULL,
  payload TEXT NOT NULL,
  created_at TEXT NOT NULL
);

CREATE TABLE crystallized_segments (
  clip_id TEXT NOT NULL,
  project_id TEXT NOT NULL,
  crystallized_at TEXT NOT NULL,
  PRIMARY KEY (project_id, clip_id)
);
```

## 8. 风险

| 风险 | 对策 |
|------|------|
| 用户以为导出=结束 | 强文案 + 继续沸腾 |
| snapshot 太大 | 只存 clips JSON，媒体引用路径 |

**降级**：跳过 precheck，仅 snapshot 文案。
