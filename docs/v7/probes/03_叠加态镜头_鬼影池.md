# 探针 03 — 叠加态镜头：鬼影池

> **概念锚点**：未敲定镜头并行存在；观测 = 坍缩，不是失败重试。  
> **实验 E2**：同一镜头的 `regenerate` 次数下降 ≥30%。

---

## 一、交互稿

### 1.1 对象定义

**叠加态单元**绑定一个「镜头槽位」`shotSlotId`（可与分镜格 / 视频节点 1:1）。

| 状态 | 含义 |
|------|------|
| `superposed` | ≥2 个候选并存 |
| `collapsed` | 已选定主版本 |
| `crystallized` | 已锁定进时间线/export（探针 10） |

### 1.2 时间轴上的概率云

- **主版本**：100% 不透明度
- **鬼影**：15–35% 不透明度，轻微横向抖动（±2px sin）
- Scrub 时鬼影**不挡**主版本，仅在暂停时显现

### 1.3 扇形展开（观测交互）

**触发**：悬停缩略图 300ms 或按 `Space` 预览时按 `Tab`

```
         ╱ 候选C 25% ╲
        ╱  候选B 30%  ╲
       │   候选A 主    │
        ╲  候选D 15% ╱
         ╲___________╱
```

- 扇区面积 ∝ `probability`（默认均匀，用户上调后记住）
- 点击扇区 → **坍缩**为该主版本，其余降为鬼影或归档
- `X` 键：删除当前扇区（负熵联动，探针 07）
- `Esc`：收起扇形，保持叠加

### 1.4 生成行为变化

- 点「生成」→ 默认**追加候选**到叠加池，**不覆盖**上一候选
- 设置项：「坍缩后生成」恢复旧行为
- 池上限默认 4（可配）；超出时提示归档最弱鬼影

## 二、数据模型

```typescript
interface ShotCandidate {
  id: string
  shotSlotId: string
  assetPath: string
  thumbPath: string
  probability: number        // 0-1，UI 权重
  isPrimary: boolean
  promptSnapshot: string
  resonanceSnapshotId?: string
  createdAt: string
}

interface SuperposedShot {
  shotSlotId: string
  projectId: string
  state: 'superposed' | 'collapsed' | 'crystallized'
  candidates: ShotCandidate[]
  primaryId: string | null
}
```

**SQLite**

```sql
CREATE TABLE shot_candidates (
  id TEXT PRIMARY KEY,
  shot_slot_id TEXT NOT NULL,
  project_id TEXT NOT NULL,
  asset_path TEXT NOT NULL,
  thumb_path TEXT NOT NULL,
  probability REAL NOT NULL DEFAULT 0.25,
  is_primary INTEGER NOT NULL DEFAULT 0,
  prompt_snapshot TEXT,
  created_at TEXT NOT NULL
);
CREATE INDEX idx_candidates_slot ON shot_candidates(shot_slot_id);
```

`shotSlotId` 生成规则：`storyboard:{groupId}:{cellIndex}` 或 `videoNode:{nodeId}`。

## 三、技术探针

### 3.1 MVP（3 天）

1. **仅支持视频节点 + 分镜格**两种 slot
2. 生成完成回调里 `appendCandidate` 而非 `setNodeData.output`
3. `SuperposedFilmstrip.tsx` 叠在 `VideoGenerator` 预览区
4. 坍缩时写回 `canvasStore` 节点 `outputUrl`，并标记 `collapsed`
