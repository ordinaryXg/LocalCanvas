# 探针 06 — 内在合唱团：声部合议

> **概念锚点**：外化自我的六个声部；AI 让声音被听见，用户拍板。  
> **假设**：比职业 Agent 角色更贴近个人创作纠结。

---

## 一、交互稿

### 1.1 六个声部

| ID | 名称 | 默认音量 | 典型发言 |
|----|------|----------|----------|
| `impulse` | 冲动者 | 0.5 | 「现在就爆炸！」 |
| `skeptic` | 怀疑者 | 0.5 | 「太俗，像网大」 |
| `nostalgic` | 怀旧者 | 0.4 | 「像小时候那部片」 |
| `perfectionist` | 完美主义者 | 0.6 | 「手型不对」 |
| `bystander` | 旁观者 | 0.4 | 「观众会走神」 |
| `dormant` | 沉睡者 | 0.2 | 「…（模糊方向）」 |

### 1.2 合议 UI：混音台 + 台词轨

```
  冲动者  ████████░░  0.8
  怀疑者  ████░░░░░░  0.4
  ...

  ── 台词轨 ─────────────────────────────
  怀疑者：赛博浓度太高，和「雨夜心悸」冲突。
  怀旧者：可保留霓虹但降饱和，像记忆里的颜色。
  沉睡者：……还有一条更静的路。

  [ 让怀疑者当家 ]  [ 手动平衡 ]  [ 开始生成 ]
```

**「让 X 当家」**：该声部音量 →1，对立声部 →0.2，持续本次会话。

### 1.3 合议触发

| 入口 | 场景 |
|------|------|
| `/deliberate` | 主动发起 |
| 共鸣场变更后 | 自动短合议（可关） |
| 悬崖建议前 | 旁观者优先发言 |

### 1.4 输出不是 WorkflowPlan

合议输出 **`ChorusResolution`**：

```typescript
interface ChorusResolution {
  tuningAdjustments: { sourceId: string; gravityDelta: number }[]
  affectAdjustments?: { timeSec: number; arousalDelta: number }[]
  promptModifiers: string[]
  blockers: string[]          // 若怀疑者音量高，可能建议「先生成静帧」
}
```

由 `chorusCompiler` 应用到共鸣场/地形，**不直接画节点**（节点由探针 12 编译）。

## 二、数据模型

```typescript
interface ChorusVoiceState {
  projectId: string
  volumes: Record<VoiceId, number>
  dominantVoice: VoiceId | null
  lastSessionId: string
}

interface ChorusUtterance {
  sessionId: string
  voiceId: VoiceId
  text: string
  stance: 'propose' | 'oppose' | 'amend'
  createdAt: string
}
```

持久化 `chorus_utterances` 供回放；`chorus_voice_state` 存音量。
